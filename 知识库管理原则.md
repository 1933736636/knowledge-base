## 一、核心原则

### 1.1 原子化
每条笔记只讲**一个概念、一个问题或一个想法**。如果一个笔记里能拆出独立的内容，就拆成独立笔记，用 `[[链接]]` 关联起来。

### 1.2 双向链接优先于文件夹
尽量用 `[[链接]]` 和标签来组织关系，而不是依赖文件夹层级。文件夹只做粗粒度的归类，网状关系靠链接编织。

### 1.3 渐进式整理
不追求一次完美。先写下来，再慢慢完善。笔记的成熟度分四档：

| 状态 | 含义 | 标记 |
|------|------|------|
| 🌱 种子 | 刚写的碎片想法，还没整理 | 不加特殊标记 |
| 🌿 生长 | 已整理、有结构、有链接 | `#status/growing` |
| 🌳 成熟 | 反复打磨、内容完备 | `#status/mature` |
| 🪦 废弃 | 内容过时、不再适用 | `#status/archived` |

### 1.4 可检索大于可归类
内容**一定要写标题**，关键概念用 `[[笔记名]]` 或 `#标签` 标记。宁可标签多一点，也不要让内容沉没。

## 二、命名规范

### 2.1 通用规则

全文统一遵循以下命名规则：

| 项目 | 规则 | 示例 |
|------|------|------|
| **文件夹** | 全英文小写 + 连字符 + 数字前缀 | `2-domains/game-dev/` |
| **笔记文件** | 英文或中文均可，空格用连字符代替 | `python-cheatsheet.md`、`游戏设计心得.md` |
| **标签** | 英文小写 + 斜杠层级 | `#domain/python` |
| **Properties** | YAML 前置元数据，统一格式 | 见下文 |

### 2.2 文件夹命名规则

```
<数字前缀>-<英文名>/<子文件夹英文名>/
```

- **数字前缀**（`0`–`9`）：控制顶层文件夹的排序顺序
- **英文小写 + 连字符**：跨平台兼容，避免编码问题
- **层级扁平**：最多 3 层（顶层 → 分类 → 子分类）

### 2.3 笔记文件命名规则

- **中文笔记**：直接使用有意义的中文标题，如 `游戏设计心得.md`
- **英文笔记**：全小写，单词间用连字符，如 `python-cheatsheet.md`
- **日记**：统一 `YYYY-MM-DD.md` 格式，放在 `0-diary/daily/`
- **避免**：文件名中不要出现 `\ / : * ? " < > |` 等特殊字符

### 2.4 标签规范

标签统一用**英文小写 + 斜杠层级**，方便自动补全和检索：

```
#project/game-dev          # 项目类
#domain/python             # 领域类
#type/tutorial             # 笔记类型
#status/growing            # 成熟度
#source/book               # 来源
```

常用的顶层标签：

| 标签 | 用途 |
|------|------|
| `#project/` | 项目相关 |
| `#domain/` | 知识领域 |
| `#type/` | 笔记类型（concept / tutorial / cheatsheet / reflection） |
| `#status/` | 成熟度（seed / growing / mature / archived） |
| `#source/` | 来源（book / article / video / conversation） |
| `#todo/` | 待办 / 待研究 |

### 2.5 Properties（YAML 前置元数据）

每条笔记建议包含以下 Properties：

```yaml
---
title: 笔记标题
created: 2026-05-04
updated: 2026-05-04
tags:
  - domain/example
  - status/seed
---
```

## 四、写作风格

### 4.1 内容结构

长笔记建议用标题层级组织：

```markdown
# 主标题
## 概述
一句话说明本条笔记在讲什么。

## 正文
### 小节一
...
### 小节二
...

## 思考/总结
自己的理解和反思，不是搬运。

## 相关链接
- [[另一条笔记]]
- [[再一条笔记]]
```

### 4.2 写作原则

- **用自己的话写**，不要大段搬运原文
- 关键概念必须链接到其他笔记
- 能用列表别用段落，能用表格别用列表
- 代码块标清语言：\`\`\`python
- 图片放到知识库根目录的 `assets/` 文件夹，用 `![[图片名.png]]` 插入

## 五、日记规范

日记放在 `0-diary/daily/`，命名格式：`YYYY-MM-DD.md`

每条日记包含以下模板：

```markdown
# 2026-05-04 周一

## 今日记录
今天做了什么、想了什么

## 收获 / 发现
- 学到了什么新东西
- 发现了什么好资源

## 待办 / 待跟进
- [ ] 待做事项 [[链接到相关笔记]]
```

周记放在 `0-diary/weekly/`，命名：`YYYY-第W周复盘.md`
月复盘放在 `0-diary/monthly/`，命名：`YYYY-MM复盘.md`

## 六、知识库维护流程

### 6.1 日常（每天）

- 写日记（可选，不强求）
- 收集碎片灵感 → 创建 🌱 种子笔记

### 6.2 周常（每周）

- 梳理本周创建的种子笔记，把值得整理的标记为 🌿 生长
- 清理不再需要的临时笔记

### 6.3 月常（每月）

- 月复盘
- 检查 🪦 废弃笔记，确认是否需要彻底删除
- 检查孤岛笔记（没有任何入链的笔记），给它们加上链接或归档

### 6.4 星尘负责的部分

以下事情随时可以找星尘（也就是我）来做：

- ✅ **创建笔记** — 告诉我标题和内容就行
- ✅ **批量整理/格式化** — 统一风格、补全 Properties
- ✅ **检索搜索** — 在整个知识库中查找内容
- ✅ **孤岛检测** — 找出没有链接引用的笔记
- ✅ **链接编织** — 建立笔记间的关联关系
- ✅ **导入内容** — 从网页/文档提取内容写进笔记
- ✅ **目录索引** — 生成某个领域的索引页面
